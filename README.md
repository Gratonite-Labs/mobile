# Gratonite — Mobile

The Gratonite iOS & Android app. Built with Expo SDK 54, React Native 0.81.5, and React 19. Supports voice/video rooms via LiveKit, real-time messaging via Socket.io, push notifications, and a native Liquid Glass-inspired UI.

## Prerequisites

- **Node.js** ≥ 20
- **pnpm** ≥ 10 — `npm install -g pnpm`
- **Expo CLI** — `npm install -g expo-cli` or use `npx expo`
- **EAS CLI** (for builds/submissions) — `npm install -g eas-cli`
- **Xcode** ≥ 15 with iOS 17 Simulator (for iOS)
- **Android Studio** with an AVD running API 33+ (for Android)
- The **Gratonite API** running on your local network or a dev server

## Getting Started

### 1. Clone the repo

```bash
git clone https://github.com/Gratonite-Labs/mobile.git
cd mobile
```

### 2. Install dependencies

```bash
pnpm install
```

### 3. Set up environment variables

```bash
cp .env.example .env
```

Edit `.env` with your local API URLs — see [Environment Variables](#environment-variables).

### 4. Start the Expo dev server

```bash
pnpm start
```

Then press `i` to open in iOS Simulator, `a` for Android, or scan the QR code with the Expo Go app.

## Environment Variables

Expo reads env vars from `.env` at the project root. Variables are accessed in code via `expo-constants` or `process.env`.

| Variable | Description | Example |
|----------|-------------|---------|
| `EXPO_PUBLIC_API_URL` | Gratonite API base URL | `http://192.168.1.100:4000` |
| `EXPO_PUBLIC_WS_URL` | Socket.io WebSocket URL | `http://192.168.1.100:4000` |
| `EXPO_PUBLIC_LIVEKIT_URL` | LiveKit WebSocket URL | `ws://192.168.1.100:7880` |
| `EXPO_PUBLIC_MINIO_URL` | MinIO media base URL | `http://192.168.1.100:9000` |

> **Note:** Use your machine's LAN IP address (e.g. `192.168.x.x`), not `localhost`, when connecting a physical device. Simulators can use `localhost`.

## Project Structure

```
├── App.tsx                   # Root component — navigation container + providers
├── index.js                  # Expo entry point
├── app.json                  # Expo app config (bundle ID, splash, icons)
├── eas.json                  # EAS Build profiles (development / preview / production)

├── android/                  # Native Android project (generated by `expo prebuild`)
└── src/
    ├── navigation/           # Stack and tab navigators
    ├── screens/              # One component per screen
    │   ├── auth/             # Login, register, MFA
    │   ├── home/             # Home feed
    │   ├── server/           # Server + channel views
    │   ├── dms/              # Direct message threads
    │   ├── voice/            # Voice/video room screen
    │   ├── friends/          # Friends list
    │   └── settings/         # App and account settings
    ├── components/           # Shared UI components
    ├── hooks/                # Custom React hooks (auth, socket, query)
    ├── stores/               # Zustand stores
    │   ├── auth.store.ts     # Current user + tokens
    │   └── voice.store.ts    # LiveKit room state
    ├── lib/
    │   ├── api.ts            # Typed fetch wrapper (mirrors web client)
    │   ├── socket.ts         # Socket.io client instance
    │   └── queryClient.ts    # TanStack Query client config
    └── types/                # Local TS types (re-exports @gratonite/types)
```

## Running on a Device or Simulator

### iOS Simulator (macOS only)

```bash
pnpm ios
```

Or start the dev server and press `i` in the terminal.

### Android Emulator

```bash
pnpm android
```

Make sure an AVD is running in Android Studio first (`adb devices` should list it).

### Physical Device

1. Install the **Expo Go** app from the App Store / Google Play
2. Start the server: `pnpm start`
3. Scan the QR code shown in the terminal with Expo Go

For features that require native code (LiveKit, notifications, etc.) you need a [development build](#eas-builds) instead of Expo Go.

## EAS Builds

EAS Build compiles the native app in the cloud without needing Xcode or Android Studio locally.

### Authenticate

```bash
eas login
```

### Build profiles (`eas.json`)

| Profile | Description |
|---------|-------------|
| `development` | Debug build with dev-client for local testing |
| `preview` | Internal distribution build (TestFlight / internal track) |
| `production` | Release build for App Store / Google Play |

### Build commands

```bash
# Development build (iOS)
pnpm eas:build:ios:dev

# Preview build (iOS)
pnpm eas:build:ios:preview

# Production build (iOS)
pnpm eas:build:ios:prod
```

## Submitting to the App Store

```bash
pnpm eas:submit:ios
```

This runs `eas submit --platform ios --profile production`. You will need:

- An active **Apple Developer** account
- An **App Store Connect** app record created for Gratonite
- An Apple ID and app-specific password (or API key) configured in EAS

## Troubleshooting

### Metro bundler stuck / cache issues

```bash
pnpm start --clear
```

### Native module not found after `pnpm install`

```bash
pnpm prebuild
```

This regenerates the native `ios/` and `android/` directories with all native modules properly linked.

### iOS build fails with "No bundle URL present"
- Make sure the Metro server is running
- Try clearing the Metro cache: `pnpm start --clear`

### Android build fails with Gradle errors

```bash
cd android && ./gradlew clean
cd .. && pnpm android
```

### LiveKit audio/video not working on device
- Ensure microphone and camera permissions are granted in device Settings
- LiveKit requires a physical device or a simulator with microphone access enabled

### Push notifications not received
- Push notifications don't work in Expo Go — you need a development build
- Make sure `EXPO_PUBLIC_API_URL` points to a server reachable from your device (not `localhost`)

### `expo-doctor` reports issues

```bash
pnpm doctor
```

Follow the suggestions in the output. Common fixes involve aligning Expo SDK versions.
